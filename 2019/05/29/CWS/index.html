Shortage of word-based training method in deep learning[5]
  1. Word-based data is very sparse according to Zipf’s law. Many words appears only one time in the corpus while very few words take up 50% of the occurrence. Note that a word is properly trained only when it shows up several times. This is why set a frequency bar to filter infrequent words improve performance.
  2. unrealistic to maintain a huge word-vector table so that many words are treated as OOV. And more words means more parameters which leads to overfitting.
  3. CWS is not good enough so that word-based model would bias down-stream tasks.
Method

The simplest but remarkably robust model is the maximum matching model which matching the longest word in the dict each time(greedy algorithm).

In terms statistical machine learning and Neural CWS Models, word segmentation is turned to tagging problem that tagging each char with BMES label which corresponding to beginning, middle, end of a word and word with a single char.

Usually unigram and bigram features are fed to embedding layer and then the model.

Model

[1] employs transformer to perform feature extraction with pre-train BERT word vectors as input. The extracted feature vectors are the input of domain projection layer which is used to do multi-task learning. A shared feature vector is added to domain-specific feature vectors. Finally, CRF is employed to perform tagging. Does Multi-Criteria Learning help to solve basic and phrase level segmentation?

[2] performs error analysis. OOV is a great challenge in CWS. And also inconsistency in word segnmentation also leads to problems.

SOTA model: LSTM-CRF. LSTM is used to output feature vector and CRF is employed to perform tagging.

[3] employs both word-level and char-level features to perform sequence tagging.(Note that problems like NER, CWS are sequence tagging problems). The difference is that it involves an extra LSTM to compute the information from word-level features and an extra input gate to decide what kinds of information should be incorporated into the corresponding feature vector(hidden state).

F1 against sentence length: longer sentences indicates lower accuracy in terms of word segmentation so that the performance of word-based models degrade with larger sequence lengths. But pure char-based models remain the same performance.

[4] is basically the same as [3]. The difference is that [4] employs subword encoding. For example, 中国科学院院士, [3] takes the embedding of 科学院 as lattice LSTM input while [4] takes the embedding of 科学院 and 学院 as lattice LSTM input.

[6] takes pre-traning features as extra features to the model and perform multi-task learning to improve performance. The training is based on beam search and the loss function resembles negative sampling.
