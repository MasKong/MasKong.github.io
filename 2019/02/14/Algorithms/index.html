Algorithms

Posted on 2019-02-14

A “loop invariant” is something that we maintain at every iteration of the algorithm[2].

Time complexity notation:

$T(n)$ is $O(g(n))$ if T(n) grows no faster than g(n) as n gets large. Upper Bound

$T(n)$ is $\Omega(g(n))$ if T(n) grows at least as fast as g(n) as n gets large. Lower Bound

$T(n) \text { is } \Theta(g(n))$ iff both:
$$
\begin{aligned} T(n) &=O(g(n)) \ & \text { and } \ T(n) &=\Omega(g(n)) \end{aligned}
$$

Recursion:
reduce a problem to one or more smaller problems and hand those problems to a black box.
There is absolutely no need to care about the black box. It’s worthwhile to note that if there is impossible to simplify the problem, just solve it. And there must be an end for the recursion.

The master theorem

Suppose that $a \geq 1, b>1$, and d are constants (independent of n).

Suppose $T(n)=a \cdot T\left(\frac{n}{b}\right)+O\left(n^{d}\right)$, Then
$$
T(n)=\left{\begin{array}{ll}{\mathrm{O}\left(n^{d} \log (n)\right)} & {\text { if } a=b^{d}} \ {0\left(n^{d}\right)} & {\text { if } a<b^{d}} \ {\mathrm{O}\left(n^{\log _{b}(a)}\right)} & {\text { if } a>b^{d}}\end{array}\right.
$$
Three parameters:
a : number of subproblems
b : factor by which input size shrinks
d : need to do $n^d$ work to create all the subproblems and combine their solutions.

The reason that there are there euqations for the master theorem is that the running time is depending on both the branching factor and the problem size.

If the branching factor is huge, it causes the number of problems to explode so that most work is at the bottom. If the problem size is huge, the most work is at the top of the tree.

The substitution method

It works where master theorem do not.

• Step 1: Guess the answer.
• Step 2: Prove your guess is correct. (by induction)
• Step 3: Profit.

k-select problem

It runs in $O\left(n\right)$ time for any resonable choice of pivot. If bad pivot is chosen, the worst running time is $O\left(n^{2}\right)$. Generally speaking, a random pivot is good enough. A better implementation is to choose the median of medians.(divide the array into 5 groups, choose median of each group and choose the median of the medians).

BucketSort

Put numbers into a set of buckets and then concatenate them. $O\left(n\right)$

RadixSort

Put numbers into a set of buckets. Each round sorting accoding to the least-significant digit. It’s a trade-off between space and time.

bucket sort assumes data are uniformly distributed in [0,1) so that data are uniformly assigned to each bucket.

  1. create bucket
  2. assign data to buckets [0.1~0.2) to the 1 bucket and [0.2~0.3) to the 2 buckets
  3. sort data in each buckets
  4. merge

Divide and Conquer

Karatsuba algorithm:

Turn one n-digit multiplication problem to three $\frac{n}{2}$ digit multiplication problem and so on and so forth. Therefore the time complexity is $3^{\log _{2} n}=n^{\log _{2} 3} \approx n^{1.6}$.

Minimum Spanning Tree(MST)

Application: circuit design, path routing…..

A tree is a connected graph with no cycles!

A spanning tree is a tree that connects all of the vertices.

The cost of a spanning tree is the sum of the weights on the edges.

Prim’s algorithm(a greedy algorithm)

greedily adding the edge with the minimum cost to the MST.

Correctness is proven by induction.

Kruskal’s algorithm

Greedily growing a forest and merging them.

At beginning, each vertix is a single tree. Find the edge with the minimum cost and merge two trees. If the edge would cause a loop, that means it is not merging two trees but connecting two nodes in a tree.

Shortest Paths(single-source shortest paths)

Shortest Path problem is usually discussed in the context of directed graph.

Unweighted Graphs: Breadth-First Search
Directed Acyclic Graphs: Depth-First Search

O(V + E) time because only a number of edges need to be visited if there are some edges from vertex u to vertex v.

It’s a kind of dynamic programming algorithm that visit vertices in topological sorting order. Note that the update could be edges that going in or edges that going out[3] [chp 8.5].

Best-First: Dijkstra’s Algorithm

It’s basically the same as BFS except that the queue is now a priority queue. The key is the distance.

In Dijkstra’s Algorithm, each vertex is scanned at most once, and thus that each edge is relaxed at most once. However, unlike in breadth-first search, each distance label dist(v) can change multiple times.

BellmanFord

BellmanFord(s): given a directed graph, find the shortest paths from the node s to every other nodes in the graph.

Time Complexity: O(VE). The algortirhm iterates V-1 times because there are V-1 nodes in the graph which guarantees that every other node is reachable. Given an iteration, for each edge that is from u to v, relax the edge if it is tense.

Works fine with negative edges.

can detect negative cycles by running the algorithm one more time after V iterations. If the values change, there is negative cycle.

Floyd-Warshall algorithm (All-Pairs Shortest Paths)

A kind of dynamic programming.

String Matching

Trie

Each trie node stores pointers to its children.

Aho-Corasick Algorithm(AC自动机)[1]

Perform string matching “in parallel” in terms of each pattern string. It is output-sensitive algorithms whose runtime depends on how much output is generated .

Great for the case where the patterns are fixed and the text to search changes

Suffix links: a link from string a to string b such that b is the longest proper suffix of a in the trie.

output links: links that pointing to pattern strings to be output. Output links from a node actually form a linked list and all node in the linked list is supposed to be output.

The Net Complexity

$\Theta(n)$ to build the trie

$\mathrm{O}(n)$ to fill in suffix links(actually 2n at most) and $\mathrm{O}(n)$ to fill in output links

Suffix tree

Great for the case where the string is fixed and the patterns to search change

each internal node in the suffix tree has at least two children because at least \$ and next character would be the children.

Time complexity of match z pattern strings is $\mathrm{O}(z)$ because z paths are required to follow and simply following path take $\mathrm{O}(1)$.

it is a “reverse” Aho-Corasick automata because string is processed.

Application: Longest Repeated Substring

Generalized suffix tree:

multiple strings are in a suffix tree. perform matching in parallel for multiple strings and multiple patterns.

Application:

  1. Longest Common Substring
  2. Longest Common Extensions
  3. Longest Palindromic Substring

Suffix trees support

  1. efficient detection of all matching substrings,
  2. efficient detection of duplicated substrings,
  3. efficient detection of common substrings,
  4. efficient detection of common extensions,
    and a lot more!

Suffix trees use space $\Theta(m)$, but with a huge
hidden constant factor.
